# Assignment 2: Market Analysis — Notes

## My Venture Idea (from Assignment 1)

An AI-powered documentation synthesis tool for R&D developers.

- Developers "word vomit" raw thoughts/notes into the tool
- The tool synthesizes input with existing documentation
- Output includes hyperlinks, appendices, and structured readable docs
- Target users: developers and R&D teams

---

## Task Mapping

### Task 1 — Market Description (100–150 words, 2 marks)
**Market:** Technical documentation software / developer productivity / knowledge management tools

**Competitors:**
- Confluence, Notion, Coda — wikis and collaborative docs
- Swimm, Mintlify — code-linked documentation
- Otter.ai, Fireflies — voice-to-text / meeting notes
- GitHub Copilot for docs, Stenography — AI doc generation

---

### Task 2 — Core Business Process (150–250 words + diagram, 6 marks)
Typical company process (e.g. Confluence):
1. User creates a page
2. Manually adds content
3. Links to other pages/appendices
4. Team reviews
5. Docs go stale over time

Diagram: flowchart of above steps

---

### Task 3 — Porter's Five Forces (300–500 words, 6 marks)
- **Rivalry:** High — Notion, Confluence, Google Docs all compete
- **Buyer power:** High — easy to switch tools
- **Supplier power:** Low-moderate
- **Threat of new entrants:** Moderate — low barrier but strong incumbents
- **Threat of substitutes:** High — big tech bundling (Microsoft, Google, Atlassian)

---

### Task 4 — System of Market Forces / CLD (150–250 words + diagram, 7 marks)
Key variables:
- Documentation quality → developer onboarding speed → team productivity → investment in tooling → documentation quality (reinforcing loop)
- Doc staleness
- Adoption friction
- Number of contributors

---

### Task 5 — Value Proposition (200–300 words, 4 marks)
**Customer pain:** Developers hate writing docs → docs are incomplete, outdated, or never written

**Voice of the Customer:** R&D teams need docs but lack time/motivation to write them properly

**Value Proposition:** Remove friction by accepting unstructured input and automatically synthesizing it into structured, linked, readable documentation
